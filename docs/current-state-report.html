Section 03

Gaps and Issues

Critical — AEO & Schema

✕
No Schema Markup on Any Listing Page

None of the current listing pages include LocalBusiness, GeoCoordinates, OpeningHoursSpecification, or FAQPage schema. This is the single biggest gap for AI search visibility. Without schema markup, AI engines like ChatGPT, Perplexity, and Google AI Overviews cannot reliably extract and cite business information. Google's own documentation confirms structured data is a primary factor in AI Overview inclusion.

✕
No FAQ Sections on Listings

FAQPage schema is one of the highest-value AEO signals available. When someone asks "Is there a dentist in Pāpāmoa that does braces?" — a listing with a structured FAQ section answering that question will be cited by AI. Currently no listing on the site has this. It is a core feature of the proposed Gold tier and will immediately differentiate paid listings from existing free ones.

✕
No LLMs.txt File

An LLMs.txt file tells AI crawlers (Perplexity, Common Crawl, and others) which parts of the site to index and prioritise. Without it, AI engines make their own decisions about what to include. Platforms like Perplexity have begun honouring this file. Early adoption gives the directory more control over how its content appears in AI-generated answers.

Significant — Content & Voice

!
Listings Written in First-Person Business Voice

Current listings use "our clinic," "we offer," "you will be treated" — self-promotional language written from the business's own perspective. AI engines weight third-party editorial content as more authoritative than self-promotion. All paid listings going forward should be written from the directory's perspective: "they offer," "the clinic provides," "customers can expect." This also creates consistency across all listings, which strengthens site-wide SEO cohesion.

!
Weak Page Title Tags

Page titles are currently formatted as "BUSINESS NAME | Papamoa Information" — no keywords, no service descriptor, no location context beyond the site name. A title like "Pāpāmoa Orthodontist — Braces & Aligners in Pāpāmoa, NZ | Papamoa.info" performs significantly better in both Google search and AI citation. Every paid listing should have a unique, keyword-rich title generated as part of the listing build.

!
Inconsistent Content Depth Across Listings

Some listings are detailed and well-structured (Orthodontist). Others are minimal placeholders. This inconsistency weakens the overall site authority. A paid tier system with mandatory content standards — minimum word count, required sections, schema markup — brings every new listing to a consistent quality floor and gradually raises the site-wide average.

Moderate — Technical

!
Contact Forms Require JavaScript

The "You must have JavaScript enabled to use this form" message on listing pages means some crawlers and accessibility tools cannot interact with contact forms. Replacing with Web3Forms provides a JS-independent, accessible form solution that also integrates with the CRM and lead tracking system.

!
No Robots.txt Review Completed

The robots.txt file has not been reviewed as part of this audit. If key listing pages or category pages are inadvertently blocked, AI crawlers cannot access them regardless of content quality. This should be reviewed and updated to explicitly allow AI crawlers access to all commercial listing pages.

i
Google Ads Conversion Pixel Active Site-Wide

A Google Ads conversion tracking pixel is embedded on every page, indicating active ad spend driving traffic to the site. This is useful context — it means the site already has measurable inbound traffic. Connecting Google Analytics and Search Console will quantify this and provide the data needed for ROI reporting to listed businesses.

i
No Commercial Monetisation System

The site currently has no paid listing tiers, no signup process, no payment system, and no CRM. The "List your business" link in the footer goes to a generic contact form. The new system addresses this in full — this is noted here as context for the scope of change proposed.

Section 06

Recommendations

The following recommendations are prioritised by impact on AI search visibility, commercial performance, and alignment with the new paid listing system.

Priority 1 — Immediate Technical Fixes

High
Add LocalBusiness JSON-LD Schema to All Listing Pages

Implement structured data markup on every listing page including business name, address, phone, website, opening hours, geo-coordinates, and category. This is the single highest-impact AEO improvement available and can be templated so every new paid listing generates it automatically.

Impact: AI citation visibility, Google rich results, ChatGPT & Perplexity indexing
High
Create and Configure LLMs.txt

Add an LLMs.txt file to the root of the domain directing AI crawlers to prioritise listing pages and category pages. Explicitly allow Perplexity, Common Crawl, and Google crawlers access to all commercial content. This is a quick technical change with outsized AI visibility impact.

Impact: AI crawler access, Perplexity citation, Google AI Overview inclusion
High
Review and Update Robots.txt

Audit the current robots.txt to ensure no commercial listing or category pages are blocked. Add explicit Allow directives for known AI crawlers (GPTBot, PerplexityBot, ClaudeBot). Disallow only low-value pages such as admin, thank-you, and test pages.

Impact: Ensures all listing content is accessible to AI and search crawlers
High
Replace JS-Dependent Contact Forms with Web3Forms

Replace the current JavaScript-required contact forms on listing pages with Web3Forms integration. This removes the accessibility barrier, enables form submissions to feed directly into the CRM, and supports lead tracking through the funnel. Free tier supports up to 250 submissions per month.

Impact: Lead capture, CRM integration, accessibility compliance

Priority 2 — Content & Voice

High
Establish Third-Party Directory Voice Standard

All new paid listings must be written from the directory's editorial perspective — "they offer," "the business provides," "customers can expect" — never first-person business voice. This positions each listing as independent editorial content rather than self-promotion, which AI engines weight as significantly more authoritative. Apply retrospectively to existing high-value listings.

Impact: AI citation authority, SEO cohesion, perceived editorial credibility
High
Update Page Title Tags to Include Keywords

Change listing page titles from "BUSINESS NAME | Papamoa Information" to a keyword-rich format: "Business Name — Service Type in Pāpāmoa, NZ | Papamoa.info". This improves click-through rates from search results and gives AI engines clearer topical signals about each page.

Impact: SERP click-through, AI topical relevance, search ranking
Medium
Add FAQ Sections to Gold Tier Listings

Every Gold listing should include a structured FAQ section of 3–5 questions and answers, formatted with FAQPage schema. Questions should mirror real search queries ("Is there a [service] in Pāpāmoa?", "How much does [service] cost in Pāpāmoa?"). This is the primary mechanism for AI citation and should be generated automatically by Claude as part of the listing build workflow.

Impact: Direct AI citation, Google AI Overview inclusion, featured snippet capture

Priority 3 — Commercial System

Launch Paid Listing Tiers (Silver & Gold)

Enable GA4 and Search Console to track listing page performance, category page traffic, and search query data. This data directly supports the ROI reporting promised to paid listing clients and provides the evidence base for the 2027 ROI Guarantee. Request access from the site owner as a priority.

Medium
Investigate Drupal REST API for Automated Listing Creation

The site runs on Drupal, which has native REST API capability. Enabling this would allow listing content to be published automatically via the sales funnel automation, removing the manual CMS step entirely. Engage Klixo to confirm API availability and document the listing node structure for integration planning.

Create a dedicated "Papamoa Beach Tide Times" page using a free tide API. "Papamoa Beach tide times today" is a high-frequency local search with no strong existing local result.
